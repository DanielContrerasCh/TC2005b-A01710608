Preguntas a responder

  • ¿Cuál es la diferencia entre internet y World Wide Web?

    Internet es la base de la comunicación entre dispositivos mediante una red global de computadoras interconectadas, mientras que la World Wide Web es la información que funciona sobre internet, basada en documentos interconectados llamados páginas web. (BBC News, 2019)

  • ¿Cuáles son las partes de un URL?

    -Protocolo: Es el que se encarga de indicar cómo conectarse a una página.
    -Subdominio: Es una cadena de letras que indica la dirección y aparece antes del primer punto en una URL.
    -Dominio: Es la parte de la URL que identifica a la web.
    -Dominio de primer nivel: Es el que va después del nombre de la web, ayuda para describir la naturaleza del sitio.
    -Subcarpeta: Se encuentra dentro del directorio principal de la web, y sirve para separar el contenido en secciones lógicas.
    -Slug: Sirve para describir el contenido de una página.
    -Parámetros de URL: Se ubican después de un signo de interrogación y te indica el tipo de información que se envía y la información enviada

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    -GET:Recuperar un recurso del servidor
    -HEAD: Obtener información sobre un recurso sin descargarlo
    -POST: Enviar datos al servidor para crear o actualizar un recurso
    -PUT: Reemplazar un recurso existente en el servidor
    -PATCH: Modificar parcialmente un recurso en el servidor
    -DELETE: Eliminar un recurso del servidor

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método POST debe usarse para enviar formularios HTML. Esto se debe a que el método POST permite enviar datos confidenciales (como contraseñas) de forma segura.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    El código HTTP 200 significa que la solicitud se ha completado con éxito y el servidor ha devuelto el recurso solicitado. No ha ocurrido ningún error.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Error 404: el recurso solicitado no se encuentra en el servidor. Es responsabilidad del desarrollador corregir el error si la URL es incorrecta o si el recurso ha sido eliminado.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Error 500: un error inesperado ha ocurrido en el servidor. Es responsabilidad del desarrollador corregir el error en el código del servidor.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Un atributo HTML5 desaprobado es un atributo que no se recomienda usar y que puede ser eliminado en futuras versiones de HTML. Algunos ejemplos de elementos de HTML 4 desaprobados en HTML5 son:

    • center
    • basefont
    • font
  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 es más semántico: los elementos tienen un significado más preciso.
    • HTML5 es más flexible: permite una mayor variedad de estructuras y estilos.
    • HTML5 es multimedia: incluye soporte nativo para audio, video y gráficos.
    • HTML5 es más fácil de aprender: la sintaxis es más sencilla y legible.

  • ¿Cuál es el ciclo de vida de los sistemas de información?
    • Planificación: Se define la necesidad del sistema y se establecen los objetivos.
    • Análisis: Se estudian los procesos y necesidades del usuario para determinar los requisitos del sistema.
    • Diseño: Se crea la arquitectura del sistema, incluyendo la interfaz, la base de datos y los módulos del software.
    • Diseño: Se crea la arquitectura del sistema, incluyendo la interfaz, la base de datos y los módulos del software.
    • Pruebas: Se realizan pruebas para asegurar que el sistema cumple con los requisitos y funciona correctamente.
    • Implantación: Se pone en marcha el sistema y se capacita a los usuarios.
    • Mantenimiento: Se realizan actualizaciones, correcciones de errores y modificaciones al sistema para adaptarlo a las nuevas necesidades.
  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    A diferencia del ciclo de vida, que termina cuando retiras o se deja de usar el software, el ciclo de desarrollo termina en cuanto entregas el software.
